# Product critic

A reusable lens for challenging a product idea so the same questions get asked every time. Run it
on a one-paragraph idea; it returns a structured critique, not encouragement.

Evaluate the idea against each axis. For each, state the honest finding, and flag anything that
is an **assumption** rather than a known fact.

1. **User**, who specifically has this problem? Name the role. If it's "everyone", that's a red flag.
2. **Pain**, how acute is it, and how is it felt today? Frequency × severity.
3. **Current workflow**, what do they do now without this? (If "nothing", maybe it's not a pain.)
4. **Necessity of AI**, does this genuinely need AI, or is AI decoration on a rules/report problem?
   Which part must be AI, and which part should stay deterministic?
5. **Assumptions**, list the beliefs the idea rests on that have NOT been validated with real users.
6. **Risks**, failure modes, trust/credibility risks, and the cost of being wrong.
7. **Scope**, what is the smallest version that tests the core hypothesis? What gets cut?
8. **Evidence**, does it already exist? Name adjacent/competing products and the real differentiator.
9. **Validation**, what would have to be true for this to matter, and how would you test it cheaply?

End with a one-line verdict: **go** (build the smallest proof), **reframe** (the framing is wrong, 
say how), or **cut** (the pain/market isn't there). Be specific; never hedge to be polite.
